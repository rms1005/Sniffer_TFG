1.-   MANUAL DE USUARIO

Este manual tiene como objeto facilitar el manejo al usuario de nuestra aplicación. A continuación detallaremos cada una de las opciones que ofrece nuestra aplicación así como su funcionalidad asociada y algunos requerimientos a la hora de introducir datos.

1.1      Introducción

La aplicación Sniffer I se concibe como una completa herramienta para la captura, filtrado, y exportación de ficheros a formato XML. A continuación veremos todas las opciones de las cuales dispone la aplicación y la forma correcta de manejarla para su más completo aprovechamiento.

 

1.1.1   Arranque de la aplicación y Menú Inicial

 

La aplicación arranca una vez que se ha hecho doble clic sobre el archivo sniffer.bat (en el caso de Windows) o se ha ejecutado el script sniffer.csh (en el caso de Linux) ambos ya comentados en el apartado de instalación de la aplicación.

 

Una vez que se ejecuta, veremos una pantalla de sistema que nos mostrará las principales opciones de ejecución de las aplicación, de igual manera, nos ofrecerá la posibilidad de selecciones la cantidad de memoria que deseamos reservar para la máquina virtual de Java y nos posibilitará abandonar la aplicación si así lo deseamos.

A continuación mostramos dicha pantalla:

Figura 20  Pantalla Inicial

1.1.2   Configuración de la Máquina Virtual de JAva

Si seleccionamos la opción 3, podremos configurar la memoria usada por la Máquina Virtual de Java.

Mostramos dicha pantalla:

Figura 21 - Pantalla de configuración MVJ

 

Una vez seleccionada la opción uno podemos introducir dicha cantidad.

Figura 22  Reserva de memoria Virtual

 

Hay que tener en cuenta que dependiendo de la memoria virtual que se reserve, se podrá trabajar con ficheros más o menos grandes. Si se desea trabajar con ficheros mayores de 100 MB, será necesario reservar una memoria de al menos el doble del tamaño del fichero para que la aplicación sea capaz de realizar todas sus funciones.

 

Una vez introducida la memoria que se desee, se mostrará la ventana del menú inicial, desde la cual podremos o bien entrar en la aplicación o bien salir de ella.

1.1.3   Ejecución en modo Comando

Si seleccionamos la segunda opción nos permitirá ejecutar la aplicación en modo comando y nos presentará una pequeña ayuda donde podremos visualizar las principales funciones que podremos ejecutar desde este modo.

A continuación mostramos dicha pantalla:

Figura 23  Ejecución en Modo Comando

Dentro del modo comando dispondremos de la mayoría de funcionalidades que tendremos en el modo comando.

Para poder sacar el máximo partido a este modo de ejecución, será necesario generar los ficheros de parametrización desde el entorno gráfico. Una vez generados (a elección del usuario) se podrán modificar con un editor de texto. El formato de los mismos, viene especificado en la documentación del programa.

A continuación describimos varias de las opciones más frecuentes del modo comando en forma de scripts.

·         Realización de una captura (Windows)

set ruta=C:\Sniffer-I

set ejecutable=Sniffer.jar

set param=-command -scan "C:\Sniffer-I\files\parametrizacion\P_Captura.xml"

set mvm=0

cd %ruta%

set PATH=.;%PATH%

set java=java

if not '%mvm%' == '0' set java=java -Xmx%mvm%m

%java% -jar %ejecutable% %param%

 

·         Realización de una captura (Linux)

ruta="C:\Sniffer-I"

ejecutable=sniffer.jar

param="-command -scan C:\Sniffer-I\files\parametrizacion\P_captura.xml"

mvm=0

cd $ruta

#set PATH=.;%PATH%

if [[ $mvm = "0" ]]

then

 java=java

else

 java="java -Xmx"$mvm"m"

fi

$java -jar $ejecutable $param
